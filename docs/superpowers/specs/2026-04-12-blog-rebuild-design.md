# 博客重建设计文档

> 站点：randolph.pro（GitHub Pages）
> 站名：黃某人
> 副标题：痴

## 1. 项目概述

将旧 Hexo 博客（停更于 2019-03）重建为现代静态博客。内容方向从纯技术转向**文学创作为主（公众号风格的小说+杂谈）、ML/AI 技术碎碎念为辅**。设计追求独特的"纸上东风"气质——素纸极简为骨架，东方意韵作点睛。

## 2. 技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| 框架 | **Astro** (SSG) | 专为内容站设计，零 JS 默认输出，Markdown/MDX 原生支持 |
| 样式 | **Tailwind CSS** + 自定义 CSS | Tailwind 做基础布局，自定义 CSS 实现东方排版特效 |
| 搜索 | **Pagefind** | 构建时生成索引，客户端全文搜索，免费无依赖 |
| 内容管理 | **Astro Content Collections** | 类型安全的 Markdown 文章管理 |
| 部署 | **GitHub Pages** + GitHub Actions | 推送到 master 自动构建部署 |
| 域名 | **randolph.pro** | 保留现有 CNAME |

## 3. 字体

- **中文**：京华老宋体（本地字体，用户已安装；web 端需提供 woff2 子集或回退方案）
- **英文**：Josefin Sans（Google Fonts 或自托管）
- **代码**：JetBrains Mono（Google Fonts 或自托管）

### 字体回退策略

京华老宋体是非标准字体，访客浏览器中大概率没有安装。需要：
1. 制作字体的 woff2 web 字体文件（需确认字体许可证允许 web 使用）
2. 若许可证不允许 web 分发，使用回退字体栈：`"京华老宋体", "Noto Serif SC", "Source Han Serif SC", "宋体", serif`

## 4. 视觉设计

### 4.1 设计气质："纸上东风"

素纸留白的极简排版为主体框架，用东方元素作为点睛装饰：竖排标题、暖金色分隔线、传统间距韵律。整体克制现代，细节处透出东方气质。横排阅读为主，竖排做装饰性使用。

### 4.2 配色方案

**亮色模式（纸上东风）：**
- 背景：素纸暖白 `#f5f0e8`
- 正文：深灰 `#555555`
- 标题：近黑 `#3a3a3a`
- 点缀/装饰：暖金 `#c4b5a0`、`#b8a88a`
- 辅助文字：`#999999`、`#bbbbbb`
- 分隔线/边框：`#e0d8c8`、`#e8e0d0`、`#d4c8b0`

**暗色模式（纯净暗色）：**
- 背景：中性深灰 `#1b1b1f`（无色温倾向）
- 正文：浅灰 `#aaaaaa`
- 标题：近白 `#ececec`
- 辅助元素：`#666666`、`#777777`
- 分隔线：`#333333`、`#444444`
- 功能性导向，不刻意延续东方主题装饰

### 4.3 主题切换

- 导航栏提供明/暗模式切换按钮（◐ 图标）
- 尊重系统偏好设置（prefers-color-scheme）
- 用户手动选择后记住偏好（localStorage）

## 5. 页面结构

### 5.1 导航栏

所有页面共享顶部导航栏：
- 左侧：站名"黃某人" + "RANDOLPH.PRO"
- 右侧：首页 | 技术 | 阅读 | 分类 | 归档 | 关于 | 搜索 | 主题切换
- 移动端：汉堡菜单折叠

### 5.2 首页 `/`（杂志封面式）

- **封面区域**：
  - 期刊概念标识："VOL. XX · YYYY 季" 赋予更新仪式感（期号基于文学文章总数自动生成）
  - 最新文章突出展示：标题 + 摘要 + "阅读全文"按钮
  - 右上角竖排"痴"字装饰
  - 渐隐分隔线
- **近期文章列表**：
  - 最多展示 5 篇，标题 + 日期 + 一句话描述
  - "查看全部 →" 链接到归档页
- **页脚**：© + RSS + GitHub 链接

### 5.3 文章阅读页（核心体验页）

三栏布局（桌面端）：

- **左栏**：浮动目录（TOC）
  - 随滚动高亮当前章节
  - 仅长文时显示
  - 移动端隐藏或折叠到顶部

- **中栏**：正文
  - 文章头部：分类标签、标题、日期 + 字数 + 预估阅读时间
  - 章节标题：中文数字（一、二、三……）
  - 首字下沉：每章开头第一个字放大
  - 引文装饰：左侧竖线 + 淡色背景 + 引号装饰
  - 文末"完"字菱形印
  - 标签展示
  - 前/后文章导航

- **右栏**：Sidenotes 边注
  - 正文中上标数字可点击跳转/高亮对应边注
  - 边注与对应正文垂直对齐
  - 移动端降级为文末脚注

### 5.4 技术页 `/tech`

- 页面标题："技术碎碎念" / "TECH NOTES"
- 技术类文章列表：标题 + 日期 + 一句话描述 + 标签
- 仅展示技术类分类的文章

### 5.5 阅读页 `/reading`

- 页面标题："阅读" / "READING"
- 书籍卡片式布局：书影（封面图占位）+ 书名 + 作者 + 读毕日期 + 简评 + 星级评分
- 数据使用 Markdown 文件管理（与文章一致，通过 Content Collections 统一处理）

### 5.6 分类页 `/categories`

- 页面标题："分类" / "CATEGORIES"
- 网格式分类入口卡片：分类名 + 文章计数
- 分类列表：匯文、杂谈、ML/AI、工具（根据实际文章动态生成）
- 匯文为文学创作的统一分类，涵盖小说和诗歌
- 点击进入该分类下的文章列表

### 5.7 归档页 `/archives`

- 页面标题："归档" / "ARCHIVES"
- 时间线式布局：按年份分组
- 每条：日期 + 圆点 + 文章标题 + 分类标签
- 旧文章预留"迁移中"占位（2019 及更早）

### 5.8 关于页 `/about`

- 页面标题："关于" / "ABOUT"
- 居中布局：头像 + 名字 + 个人签名
- 社交链接：GitHub、Email、RSS

## 6. 交互特效

### 6.1 阅读进度条

- 页面顶部 2px 细线
- 暖金色渐变（`#c4b5a0` → `#b8a88a`）
- 随页面滚动推进
- 暗色模式下颜色适配

### 6.2 浮动目录（TOC）

- 文章页左侧固定定位
- 自动从文章 Markdown 标题生成
- 当前阅读章节高亮
- 点击可跳转
- 窄屏时隐藏

### 6.3 Sidenotes 边注

- 正文中用上标数字标记
- 对应注释在右侧边栏展示，垂直对齐
- 点击上标数字高亮对应边注
- 响应式降级：< 1200px 时变为文末脚注

### 6.4 文学感动效

- 文章首次加载：标题淡入 + 微上移（fade-in + translate-up）
- 首段文字渐显
- 动效克制，持续时间 0.4-0.6s，仅首屏触发
- 尊重用户 `prefers-reduced-motion` 设置

### 6.5 东方排版元素

- **首字下沉**：每章第一个字 float:left 放大显示
- **竖排装饰**：首页"痴"字、阅读页书影区域的竖排文字
- **渐隐分隔线**：`linear-gradient(90deg, transparent, #c4b5a0, transparent)` 取代粗横线
- **引文样式**：左侧 2px 竖线 + 淡色背景 + 引号装饰
- **"完"字菱形印**：文末收束符号，旋转 45° 的菱形框

## 7. 搜索

- 使用 **Pagefind**（Astro 官方推荐）
- 构建时自动扫描生成的 HTML，创建搜索索引
- 客户端 JS 加载索引并执行搜索
- 搜索 UI：导航栏搜索图标 → 弹出搜索对话框（modal）
- 支持中文分词搜索

## 8. RSS

- 使用 `@astrojs/rss` 官方集成
- 输出 `/rss.xml`（或 `/atom.xml`）
- 包含所有文章（文学 + 技术）

### Markdown Frontmatter 规范

**文学文章（posts）：**
```yaml
---
title: "昆虫白与舞茉莉"
date: 2026-03-15
category: "匯文"        # 匯文 | 杂谈
tags: ["小说", "创作"]
description: "一句话描述"
featured: true          # 是否在首页封面展示
draft: false
---
```

**技术文章（tech）：**
```yaml
---
title: "关于 Transformer 的碎碎念"
date: 2026-04-01
category: "ML/AI"       # ML/AI | 工具 | 前端 | ...
tags: ["ML", "NLP"]
description: "一句话描述"
draft: false
---
```

**阅读记录（reading）：**
```yaml
---
title: "百年孤独"
author: "加西亚·马尔克斯"
date: 2026-03-01        # 读毕日期
rating: 5               # 1-5 星
cover: "/images/books/hundred-years.jpg"  # 可选封面图
---
简评内容...
```

## 10. 响应式设计

| 断点 | 布局变化 |
|------|----------|
| ≥ 1200px | 三栏布局（TOC + 正文 + Sidenotes） |
| 768-1199px | 单栏（TOC 折叠到顶部，Sidenotes 变脚注） |
| < 768px | 单栏移动端（汉堡菜单，简化装饰元素） |

## 11. 部署

- **GitHub Actions** 自动化：push 到 master → Astro 构建 → 部署到 GitHub Pages
- CNAME 文件保留 `randolph.pro`
- `.gitignore` 排除 `node_modules/`、`dist/`、`.superpowers/`

## 12. 旧文章迁移

- 旧文章源文件（Markdown）在用户本地电脑中，后续拷贝后处理
- 归档页预留"2019 及更早"占位区域
- 迁移时放入 `src/content/legacy/` 目录
- 具体迁移策略待旧文件到位后讨论

## 13. 不包含的功能

- 评论系统（明确关闭）
- 多语言支持
- 后端/服务端逻辑
- 第三方搜索服务（Algolia 等）
