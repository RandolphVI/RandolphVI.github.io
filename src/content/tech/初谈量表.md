**测量的第一步，往往不是「测出一个数」，而是想清楚「这个数属于哪一类」。** 同样是数字 20，温度的 20°C 与长度的 20 米，能做的运算、能下的结论却截然不同——前者不能说「两倍热」，后者却确实是另一根棍子的「两倍长」。决定这一差异的，是数字背后的**测量尺度（measurement scale）**，也称**量表**。量表层级越高，数据承载的信息越丰富，可施加的数学运算也越自由。本文先沿史蒂文斯的经典分类梳理四类量表，再聚焦其中信息量最高的**定比量表（Ratio Scale）**。

## 测量的四个层级：史蒂文斯类型学

1946 年，心理物理学家 S. S. Stevens 提出了沿用至今的测量尺度分类，按信息量由低到高分为四级：**定类 < 定序 < 定距 < 定比**。每升高一级，都在前一级之上新增一种数学性质。

- **定类量表（Nominal Scale）**：数字仅作**类别标签**，彼此之间没有大小或先后之分，例如性别、国家代码、模型所属家族。此时数字只能回答「相同还是不同」，连排序都无从谈起。
- **定序量表（Ordinal Scale）**：类别之间有了**有意义的顺序**，但相邻等级的「间距」未必相等，例如满意度（很差／一般／很好）、比赛名次、疼痛分级。你知道第一名优于第二名，却无法断言这一差距与第二、第三名之间相同。
- **定距量表（Interval Scale）**：在排序之上，**差值也有了意义**——相邻刻度间隔处处相等；但零点是人为约定的，并不代表「完全没有」。摄氏温度是典型例子：20°C 比 10°C 高 10°C（差值有意义），可 0°C 只是水的冰点而非「没有热量」，因此不能说 20°C 是 10°C 的两倍热。
- **定比量表（Ratio Scale）**：在定距的全部性质之上，再拥有一个**绝对零点**——零真正代表该属性的「空无」。长度、时间、质量、开尔文温度、计数都属此类。唯有到了这一级，「A 是 B 的几倍」才成为一句有意义的陈述。

四级层层递进：定类只能分类，定序可以排序，定距能比较差距，定比则能比较倍数。

## 定比量表：测量的黄金标准

定比量表之所以居于测量尺度的顶端，关键在于两个相互关联的特性。

### 绝对零点（Absolute Zero）

定比量表的零点不是任意约定，而是代表该属性**完全不存在**的真实状态：

- **长度**：0 米意味着完全没有长度，棍子的长度不可能为负。
- **时间**：0 秒意味着没有任何时间流逝，持续时间不能为负。
- **质量**：0 千克意味着没有任何物质。
- **开尔文温度**：0 K 即「绝对零度」，粒子的热运动理论上完全停止，不存在更低的温度。

与之对照，摄氏温度的 0°C 只是水的冰点——一个约定俗成的参照，其下仍有温度（负数）。**零点是否「绝对」，正是定距量表与定比量表的分水岭。**

### 有意义的比率（Meaningful Ratios）

正因为存在绝对零点，定比量表上**不仅差值有意义，比率同样有意义**——你可以名正言顺地说一个值是另一个值的「多少倍」：

- 10 米的棍子确实是 5 米棍子的两倍长（10 / 5 = 2）。
- 耗时 20 秒的过程确实是 10 秒过程的两倍长。
- 2 千克的物体确实是 1 千克物体的两倍重。
- 600 K 确实是 300 K 的两倍热——在热力学意义上，其平均分子动能恰好是两倍。

而在摄氏温度上，这种倍数关系便会失效。20°C 比 10°C 高 10°C 没有问题，但若换算成开尔文：10°C = 283.15 K、20°C = 293.15 K，两者之比约为 1.035，仅高出约 3.5%，远谈不上两倍。**一旦离开绝对零点，比率就失去了物理意义。**

## 量表层级为何至关重要

区分量表绝非咬文嚼字，它直接决定了数据「能做什么」：

- **数学运算的自由度**：定比量表上的数据支持加、减、乘、除全部四则运算，且结果在概念上始终有意义。平均值、速度（距离／时间）、密度（质量／体积）之所以可靠，正因为它们建立在定比量表之上；而施加在低层级量表上的运算——例如对名次求平均——往往无意义甚至具有误导性。
- **精确的比较与建模**：在科学与工程中，「A 是 B 的两倍」「C 是 D 的三分之一」这类陈述是建模与推理的基石，而它只在定比量表上才成立。
- **结论的边界**：量表层级圈定了「哪些话能说」。在低层级量表上，「给名次求平均」「说难度翻了一倍」之类的陈述往往似是而非；唯有认清数据所属的量表，才能分清一个结论是严谨还是越界。下一节的「试题难度」便是一块绝佳的试金石。

## 试题难度：量表概念的一块试金石

把量表的视角放到「试题难度」上，会发现这个看似简单的概念，其实处处藏着量表层级的讲究。

**最朴素的难度——通过率。** 经典测验理论（Classical Test Theory, CTT）里，一道题的难度常直接用**通过率（答对人数的占比）**来表示：通过率越高，题目越简单。通过率本身是带绝对零点的比例（0% 表示无人答对），可一旦拿它当「难度尺度」，麻烦就来了。其一，它**依赖样本**——同一道题，给高手做与给新手做，通过率天差地别；其二，通过率的等量变化并不对应难度的等量变化——从 95% 降到 85%，与从 55% 降到 45%，在「题目究竟难了多少」上完全是两回事。换言之，把通过率直接当难度，更接近一个**定序量表**：能排出孰难孰易，却说不清「难多少」。

**更讲究的难度——IRT 的难度参数。** 项目反应理论（Item Response Theory, IRT）则把题目难度 $b$ 与被试能力 $\theta$ 放到**同一把标尺**上：难度 $b$ 被定义为「答对概率恰好为 50% 时所对应的能力水平」。这把对数几率（logit）标尺是一个**定距量表**——刻度间隔处处等价，因此「题目 A 比题目 B 难 2 个 logit」是一句有意义的话；但它**没有绝对零点**（$\theta = 0$ 通常只是被试群体的平均水平，是约定而非「零能力」），所以**不能说「题目 A 的难度是 B 的两倍」**。这恰是定距量表的典型局限。

**追求比率的难度——前沿评估的努力。** 也正因如此，一些前沿评估框架会刻意为难度（或与之对偶的「需求级别」）构造一个**定比量表**，让「难度加倍」重新具备意义。例如约定「需求每提高一级，被试成功的对数几率减半」，便把难度锚定在了一个可解释、带绝对零点的尺度上——这背后，正是把试题难度从「能排序」一路推向「能比倍数」的黄金标准追求。

## 四类量表对比

| 特性 | 定类（Nominal） | 定序（Ordinal） | 定距（Interval） | **定比（Ratio）** |
| --- | :---: | :---: | :---: | :---: |
| 分类 / 标签 | ✔ | ✔ | ✔ | ✔ |
| 排序 | | ✔ | ✔ | ✔ |
| 相等差值 | | | ✔ | ✔ |
| 绝对零点 | | | | ✔ |
| 有意义的比率 | | | | ✔ |
| 典型例子 | 性别、国家代码 | 名次、满意度 | 摄氏温度、智商分数 | 长度、时间、质量、开尔文温度、计数 |

一句话记忆：**定类分类，定序排序，定距比差距，定比比倍数。**

## 小结

量表是测量的隐形地基。它决定了一组数字究竟承载了多少信息、能被如何运算、又能支撑起怎样的结论。四级尺度层层叠加，到了定比量表，凭借一个代表「无」的绝对零点，「A 是 B 的多少倍」这样的说法才真正变得精确而有意义。无论是度量一根棍子的长度，还是刻画一道试题的难度，先问一句「我手上的数字，究竟属于哪一类量表」，往往比急着计算均值与排名更为关键。
